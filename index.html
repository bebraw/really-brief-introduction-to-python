Implementing "title" - Implementation 

def title(s=None):
    # idea: split, set first to upper return joined
    to_upper = lambda a: a[0].upper() + a[1:]
    return ' '.join(map(to_upper, s.split())) if s else None

assert title('Hello world!') == 'Hello World!'
assert title('hi') == 'Hi'
assert title() == None

Implementing "title" - Docs 

def title(s=None):
    ''' Converts given 's' to title format

    Example: "foo bar" -> "Foo Bar"
    '''
    to_upper = lambda a: a[0].upper() + a[1:]
    return ' '.join(map(to_upper, s.split())) if s else None

assert title('Hello world!') == 'Hello World!'
assert title('hi') == 'Hi'
assert title() == None

Anatomy of a Class 

class MyObject(object): # supports MI!
    a = 3 # class attribute

    def __init__(self, a, *args, **kwargs): # initializer
        # super(MyObject, self).__init__(a, args, kwargs)
        self.boo = 'foo' # instance attribute

    def __add__(self, other):
        pass # magic method for '+' op

    def do_something(self): # method
        pass # _ = "protected", __ = "private"

    @classmethod
    def bar(cls, a, b):
        pass # call using MyObject().foo(123, 234)

    @staticmethod
    def foo(a, b):
        pass # call using MyObject.foo(234, 123) or MyObject().foo(234, 123)

Map, filter, reduce 

>>> map(lambda a: a * 2, range(5))
[0, 2, 4, 6, 8]
>>>
>>> filter(lambda a: a > 3, range(5))
[4]
>>>
>>> reduce(lambda a, b: a + b, range(5))
10
>>> # note that reduce was moved to functools in py3k

Comprehensions 

>>> [a * 2 for a in range(5)] # works as map
[0, 2, 4, 6, 8]
>>>
>>> [a for a in range(5) if a > 3] # works as filter
[4]
>>>
>>> [a + b for a, b in zip(range(5), range(5))]
[0, 2, 4, 6, 8]
>>>
>>> (a * 2 for a in range(5))
<generator object <genexpr> at 0x10f365140>
>>>
>>> # set and dict comprehensions are included in py3k

Overview

  • Versatile, dynamically typed scripting language
  • Surprisingly powerful (batteries included)
  • Fairly popular (8th at TIOBE)
  • Easy to grasp basics, lots of features

Pros

  • High level language
  • Fast to develop
  • Easy to test
  • Well supported

Cons

  • Not so fast
  • Mediocre IDE support
  • Easy to make subtle mistakes
  • Py2 -> Py3k transition still ongoing

Pro Tips

  • Use virtualenv and virtualenvwrapper
  • Explore PyPI
  • Utilize pip and requirements.txt
  • When in doubt, REPL and test things
